# Mnemo Insurance - Wireframes

## Design Principles

1. **Non-blocking** - Users can navigate anywhere while documents process in background
2. **Unified chat** - All analysis modes (compare, compliance, gap) use similar chat interface
3. **Quick actions** - One-click access to main features from anywhere
4. **Status visibility** - Always show processing jobs in a non-intrusive way

## Summary

| Screen | Purpose |
|--------|---------|
| Dashboard | Home base, quick upload, mode buttons, recent activity |
| Upload | Drag-drop with progress, can navigate away |
| Policies List | Browse all policies grouped by insured |
| Policy Detail | Deep view of single policy with action buttons |
| Chat (Unified) | Same interface for all modes, context bar shows mode |
| Quote Comparison | Chat pre-loaded with comparison table |
| Compliance Check | Chat pre-loaded with compliance results |
| Gap Analysis | Chat pre-loaded with gap findings |
| Mode Selection | Pick policies/quotes before entering chat |

**Key UX patterns:**
- Persistent job status bar at bottom
- Mode context always visible at top of chat
- Suggested prompts to guide users
- Source citations on AI responses
- Non-blocking navigation throughout
